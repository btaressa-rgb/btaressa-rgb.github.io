URI Notation

Home > Technical > W3 Naming > Notation

The notation used for Universal Resource Identifiers (URIs) in the World Wide Web follows a formal grammar that defines how addresses are structured and interpreted by browsers and servers.

BNF Grammar

The formal notation for W3 addresses uses Backus-Naur Form (BNF) grammar. The general structure of a URI is:

URI        = scheme ":" schemepart
scheme     = ALPHA *( ALPHA | DIGIT | "+" | "-" | "." )
schemepart = *xchar | ip-schemepart

For internet-based schemes, the common format includes a host and optional port:

ip-schemepart = "//" login [ "/" urlpath ]
login         = [ user [ ":" password ] "@" ] hostport
hostport      = host [ ":" port ]
host          = hostname | hostnumber
port          = digits

Path Notation

The path component of a URI follows Unix-style hierarchical notation:

  • A leading / indicates an absolute path from the root of the server.
  • Path elements are separated by /.
  • .. refers to the parent directory.
  • . refers to the current directory.

Examples of valid path notation:

/hypertext/WWW/TheProject.html
../addressing/w3-naming.html
./relative-naming.html

Fragment Notation

A fragment identifier, introduced by #, refers to a named anchor within a document:

http://info.cern.ch/hypertext/WWW/TheProject.html#section1

The fragment is processed by the browser client and is not sent to the server.

Special Characters and Escaping

Certain characters have reserved meaning in URIs and must be percent-encoded when used in other contexts:

  • %20 — space
  • %2F — forward slash (when used literally)
  • %23 — hash (when used literally)
  • %3F — question mark (when used literally)

The encoding uses a percent sign followed by two hexadecimal digits representing the ASCII code of the character.
