Hypertext Terms

Home > Hypertext > Hypertext Terms

This is a glossary of terms used within the WorldWideWeb project. These definitions generally correspond to conventional usage within the hypertext community.

Anchor
An area within the content of a node which is the source or destination of a link. The anchor may be the whole of the node content. Typically, selecting an anchor causes the link to be followed. (Also sometimes known as "span", "region", "button", or "extent").
Annotation
The linking of a new commentary node to an existing node. If readers can annotate nodes, they can immediately provide feedback if the information is misleading, out of date, or incorrect.
Browser
A program which allows a person to read hypertext. The browser gives some means of viewing the contents of nodes and of navigating from one node to another.
Client
A program which requests services of another program. Normally, the browser is a client of a data server.
Daemon
A program which runs independently of the browser. Daemons may perform various management tasks such as building indexes, overviews, and back-links.
Document
A term for a node on some systems. Sometimes used as a term for a collection of nodes on related topics. The preferred term in W3 documentation.
Hypermedia
Multimedia hypertext. Media other than text typically include graphics, sound, and video.
Hypertext
Text which is not constrained to be linear.
Node
A unit of information. Also known as a frame or card. Used with this special meaning in hypertext circles: do not confuse with "node" meaning "network host."
Server
A program which provides a service to another, known as the client. In a hypertext system, a server will provide hypertext information to a browser.
Web
A set of nodes interconnected by links. Often, the set of all the nodes which are interconnected.
