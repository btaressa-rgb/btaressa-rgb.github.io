Documentation Ideas

Home > Executive Summary > Line Mode Browser > Documentation Ideas

Notes and ideas for improving the documentation of the WWW Line Mode Browser. This page collects suggestions from users and developers for making the browser easier to learn and use.

User Guide

The primary documentation resource for the line-mode browser is the User Guide, which covers:

  • Following hypertext references using numbered links
  • Searching index documents with the find command
  • Navigation commands (back, home, top)
  • Accessing help and further documentation

Suggested Improvements

The following areas have been identified as needing better documentation:

Command Line Options
The browser accepts several command-line flags when launched. These should be documented with examples showing how to start the browser pointing at a specific document or with specific display options.
Configuration Files
Users can customize the browser's behavior through configuration files. The location and syntax of these files should be clearly documented for each supported platform.
Protocol Support
The browser supports multiple protocols (HTTP, FTP, Gopher, NNTP). Each protocol has specific behaviors that users should understand when navigating different types of servers.
Error Messages
Common error messages should be documented with explanations of their causes and suggested remedies.
Installation Guide
Step-by-step installation instructions for each supported platform (Unix, VMS, etc.) would help new users get started more quickly.

Submitting Feedback

If you have suggestions for improving the line-mode browser documentation, please send them to www-bug@info.cern.ch. All feedback is welcome and will be considered for future documentation updates.
