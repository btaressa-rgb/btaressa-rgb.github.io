===========================================================
       1. CSS VARIABLES — DESIGN TOKENS
       ============================================================
       CSS variables (also called "custom properties") let you
       define a value once and reuse it everywhere in your CSS.

       Syntax:  --variable-name: value;
       Usage:   color: var(--variable-name);

       The :root selector targets the very top of the document,
       so these variables are available to every element on the page.

       TO RETHEME THE ENTIRE SITE: change only the hex values below.
       Every button, link, label, and bar that uses var(--accent)
       will update automatically.
    ===========================================================

       ============================================================

       and the page looks the same in every browser.

By default, CSS calculates width EXCLUDING padding and border.
        So a div with width: 300px and padding: 20px would actually
        be 340px wide — confusing and hard to work with.

        box-sizing: border-box changes this so that padding and
        border are INCLUDED in the declared width.
        A div with width: 300px and padding: 20px stays exactly
        300px wide. This makes layouts much more predictable.

Makes the page scroll smoothly when the user clicks an
        anchor link (like href="#projects") instead of jumping
        instantly. This is a small quality-of-life improvement.

font-family lists fonts in priority order — the browser
        uses the first one it finds on the user's device.

        -apple-system        → San Francisco (Mac/iPhone)
        BlinkMacSystemFont   → San Francisco (Chrome on Mac)
        "Segoe UI"           → Windows system font
        Roboto               → Android system font
        sans-serif           → any sans-serif font as final fallback

        This technique gives the page a "native" feel on every
        device without loading a web font, which speeds up the page.

Sets the page background to our cool off-white variable

Sets the default text color for the whole page.
        Child elements inherit this unless they override it.

Hides any content that overflows horizontally.
        This prevents a horizontal scrollbar from appearing
        when animated elements briefly extend past the viewport
        (like the mobile nav panel sliding in from off-screen).

line-height controls the vertical space between lines of text.
        1.65 means each line is 1.65× the font size tall.
        The browser default (1.2) is too tight for body text.
        1.65 is comfortable and easy to read.

Prevents images and videos from overflowing their container.
        If a container is 400px wide and the image is 800px,
        max-width: 100% shrinks the image to fit within 400px.

Images are inline by default, which adds a small gap below
        them (space reserved for text descenders). display: block
        removes that gap so images sit flush in their containers.

===========================================================
       12. SCROLL FADE-UP ANIMATIONS
       ============================================================
       Elements with class="fade-up" start invisible and shifted
       downward. When JavaScript detects them entering the viewport,
       it adds class="visible" which triggers the CSS transition.

       HOW IT WORKS:
         1. An element has class="fade-up" — starts invisible
         2. User scrolls down
         3. JavaScript IntersectionObserver detects the element
            entering the viewport
         4. JavaScript adds class="visible" to the element
         5. CSS transitions opacity from 0→1 and translateY from
            36px→0 simultaneously — element fades up into place
    ===========================================================

Starts invisible

Starts 36px below its final position

Both opacity and transform animate over 0.8 seconds.
        The ease curve starts slow, speeds up, then slows again —
        a natural, physical-feeling motion.

Fades fully into view

Moves to its natural position

Staggered delays for grid children — each card starts
       slightly after the previous one for a wave effect

       ============================================================

       define a value once and reuse it everywhere in your CSS.

       Syntax:  --variable-name: value;
       Usage:   color: var(--variable-name);

       so these variables are available to every element on the page.

===========================================================
       12. SCROLL FADE-UP ANIMATIONS
       ============================================================
       Elements with class="fade-up" start invisible and shifted
       downward. When JavaScript detects them entering the viewport,
       it adds class="visible" which triggers the CSS transition.

       HOW IT WORKS:
         1. An element has class="fade-up" — starts invisible
         2. User scrolls down
         3. JavaScript IntersectionObserver detects the element
            entering the viewport
         4. JavaScript adds class="visible" to the element
         5. CSS transitions opacity from 0→1 and translateY from
            36px→0 simultaneously — element fades up into place
    ===========================================================

Starts invisible

Starts 36px below its final position

Both opacity and transform animate over 0.8 seconds.
        The ease curve starts slow, speeds up, then slows again —
        a natural, physical-feeling motion.

Fades fully into view

Moves to its natural position

Staggered delays for grid children — each card starts
       slightly after the previous one for a wave effect

       Each project card's media area supports THREE swappable styles:

       Mix and match across cards. Use the inline `--thumb-grad` CSS variable
       on .thumb-gradient cards to set per-card colors without writing new CSS.

       REDUCED-MOTION OVERRIDES
       ============================================================
       This page is the quieter twin of ../projects.html. This block
       is the ONLY stylistic difference — it neutralizes the
       animations and microinteractions from the standard version.

       Disabled here:
         • All CSS transitions (color/hover changes snap, not fade)
         • All CSS keyframe animations
         • Hover transforms (card lift, image zoom, tag pop)
         • Smooth scroll jumps straight to anchors
         • Scroll-triggered .fade-up — forced visible at page load

       Kept as-is:
         • Static visuals: shadows, borders, gradients, blur
         • Layout, typography, color palette
         • Hover color/opacity changes (they just apply instantly)

       !important is used throughout so these rules win no matter
       where the original declaration lives in the stylesheet.
